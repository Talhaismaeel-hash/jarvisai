# Installation Guide

## Prerequisites

- **Node.js 18.18 or later** (Node 20 LTS recommended) — Next.js 15 requires this minimum
- **npm 9+** (comes with Node) — yarn/pnpm also work if you prefer, just swap the commands below
- A code editor (VS Code recommended, for TypeScript IntelliSense)

Check your versions:

```bash
node -v
npm -v
```

---

## 1. Get the code

If you received this as a zip file, unzip it. If it's a Git repo:

```bash
git clone <your-repo-url>
cd jarvis
```

## 2. Install dependencies

```bash
npm install
```

This installs Next.js, React, Tailwind, Framer Motion, all Radix UI primitives, lucide-react icons, react-markdown, and the Supabase client libraries (unused until configured).

If you see a peer-dependency warning related to React 19, it's safe to ignore — every package in `package.json` is pinned to a version that supports React 19.

## 3. Set up environment variables (optional for now)

```bash
cp .env.example .env.local
```

You do **not** need to fill anything in to run the app — every page works against mock data in `services/mockData.ts`. Fill in `.env.local` only when you're ready to connect:

- `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` — from your Supabase project settings → API
- `ANTHROPIC_API_KEY` — if you build an `/api/chat` route backed by the Anthropic API

## 4. Run the dev server

```bash
npm run dev
```

Visit **http://localhost:3000**.

- `/` — landing page
- `/login`, `/register` — auth pages (sign-in is stubbed; submitting redirects to the dashboard after a short simulated delay)
- `/dashboard` — main app (sidebar nav to Chat, Documents, Tasks, Memory, Agents, Settings)

## 5. Type-check and lint (recommended before committing)

```bash
npm run type-check
npm run lint
```

## Troubleshooting

**Port 3000 already in use**
```bash
npm run dev -- -p 3001
```

**Fonts look like fallback system fonts**
The first `npm run dev` needs internet access once to download Space Grotesk / Inter / JetBrains Mono via `next/font/google`. After that they're cached locally — no runtime calls to Google Fonts ever happen in production.

**TypeScript errors about missing `next-env.d.ts` content**
This file is auto-generated by Next.js on first run and is fine to ignore/overwrite — it's listed in `.gitignore` for that reason.

**Styling looks broken / unstyled**
Make sure `npm install` finished without errors — Tailwind needs `tailwindcss`, `postcss`, and `autoprefixer` from devDependencies to process `styles/globals.css`.
